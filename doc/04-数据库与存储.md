# 数据库与存储

## 1. 方案概览

- **db 层**：只做连接与会话（`base`、`engine`、`session`），不包含业务表与仓库。
- **领域层**：`app/domain/` 下放模型（表定义）与仓库（数据访问），依赖 `app.db` 的 Base 与 `get_session`。
- **工具层**：`agents/tools/db.py` 只做「工具该做的事」——调用领域层仓库或只读能力（如 list_db_tables 用 engine），不直接写 SQL 或放业务模型。
- **生命周期**：启动时先导入领域 models（注册到 `Base.metadata`），再 `init_db()` 建表；关闭时 `dispose_db()` 释放连接池。

## 2. 目录结构

```
app/db/                      # 数据库层：仅连接、会话、基类
├── __init__.py
├── base.py                  # SQLAlchemy DeclarativeBase
├── engine.py                 # get_engine、init_db、dispose_db
└── session.py               # get_session_factory、get_session

app/domain/                   # 领域层：模型与仓库
├── __init__.py
├── models/
│   ├── __init__.py
│   └── example.py            # Example 表（继承 app.db.base.Base）
└── repositories/
    ├── __init__.py
    └── example_repo.py       # ExampleRepo（只封装查询/写入，不自己管理 session）

app/agents/tools/
└── db.py                     # list_db_tables（用 get_engine）、query_examples（用 ExampleRepo）
```

- **db**：无 models、无 repositories，只提供 Base、引擎、会话与 init/dispose。
- **domain**：表定义与按实体的 CRUD 封装；仓库方法统一由调用方传入 Session。
- **tools/db**：只读、安全地调用领域仓库或 engine，供 Agent 使用。

## 3. 配置

| 变量 | 默认 | 说明 |
|------|------|------|
| `DATABASE_URL` | `sqlite:///./data/app.db` | 数据库 URL。PostgreSQL 示例：`postgresql://user:pass@host/dbname` |

SQLite 时 `init_db()` 会按 URL 路径创建目录（如 `data/`）。

## 4. 使用方式

### 路由或服务中（写操作）

```python
from app.db import get_session
from app.domain.repositories import ExampleRepo

# 只读
with get_session() as session:
    items = ExampleRepo.list_all(session, limit=10)

# 写操作
with get_session() as session:
    ExampleRepo.create(session, title="标题", content="内容")
```

### Agent 工具（只读）

- **list_db_tables**：用 `get_engine()` + inspect 列出表名，返回 JSON。
- **query_examples**：打开只读 session 后调用 `ExampleRepo.list_all(session, limit)`，返回 JSON。

工具在 `app/agents/tools/db.py`，已注册到 `AGENT_TOOLS`；未配置或出错时返回 `{"error": "..."}`。

## 5. 扩展

- **新表**：在 `app/domain/models/` 下新增模块，继承 `app.db.base.Base`，并在 `domain.models.__init__` 中导出；启动时导入 `app.domain.models` 后 `init_db()` 会建表。
- **新仓库**：在 `app/domain/repositories/` 下新增 `XxxRepo`，方法统一接收调用方传入的 Session。
- **新 DB 工具**：在 `app/agents/tools/db.py` 增加 `@tool`，在 `tools/__init__.py` 注册；仅做只读、白名单表或固定查询。
- **换数据库**：改 `DATABASE_URL`，安装对应驱动（如 `psycopg2-binary`、`pymysql`）。

## 6. MySQL 连接与 Demo

### 驱动

已支持 MySQL，需安装：

```bash
pip install pymysql cryptography
```

（`requirements.txt` 已包含。）

### 连接 URL

在 `.env` 中设置：

```env
DATABASE_URL=mysql+pymysql://用户:密码@主机:端口/库名
```

示例：`mysql+pymysql://root:secret@127.0.0.1:3306/ag_db`

- 请先在 MySQL 中创建好库（如 `CREATE DATABASE ag_db;`）。
- 端口省略时默认为 3306。

### 运行 Demo 脚本

在项目根目录执行（会建表、插入一条 Example、查询并打印、释放连接）：

```bash
# 使用 .env 中的 DATABASE_URL（含 MySQL）
python -m scripts.mysql_demo

# 或临时指定 MySQL
export DATABASE_URL="mysql+pymysql://root:密码@127.0.0.1:3306/ag_db"
python -m scripts.mysql_demo
```

不设置 `DATABASE_URL` 时使用默认 SQLite（`./data/app.db`）。Demo 脚本位置：`scripts/mysql_demo.py`。

## 7. 安全约定

- Agent 可调用的 DB 工具仅**只读**、**白名单表/固定查询**（如 `query_examples`）。
- 不在工具中拼接用户输入为原始 SQL；写操作只在领域仓库层、由路由或服务在受控会话中调用。
