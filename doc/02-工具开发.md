# 工具开发

## 1. 定位

- **工具**：Agent 可调用的能力封装，带名称与参数 schema，由 LLM 决定是否调用。
- **实现**：某领域的具体逻辑（如检索）；可与对应 `@tool` 放在同一模块（如 `tools/retrieval.py` 中既有 `retriever()` 又有 `retrieve_documents`）。

## 2. 目录约定

```
app/agents/tools/
├── __init__.py      # 统一注册 AGENT_TOOLS、get_agent_tools、get_tools_by_name
├── retrieval.py     # 检索：retriever() + @tool retrieve_documents
├── time_tools.py    # 时间：get_current_time
├── common.py        # 通用：echo
└── （按需）search.py、notify.py 等
```

- 按**领域**一个文件（或一类工具一个文件），单文件内聚。
- `__init__.py` 只做导入与注册，不写业务逻辑。

## 3. 新增工具（已有模块）

1. 在对应模块（如 `common.py`）中增加 `@tool` 函数，写好 docstring 与参数类型。
2. 在 `tools/__init__.py` 中导入并加入 `AGENT_TOOLS` 与 `__all__`。

无需修改 `agent.py`，Agent 通过 `get_agent_tools()` / `get_tools_by_name()` 自动包含新工具。

## 4. 新增领域模块

1. 新建文件，如 `tools/search.py`，定义 `@tool`（及可选底层函数）。
2. 在 `tools/__init__.py` 中导入并加入 `AGENT_TOOLS` 与 `__all__`。

## 5. 规范

- 工具返回 `str` 或可 `json.dumps` 的结构，便于写回 `ToolMessage`。
- docstring 与 Args 会生成 schema 供 LLM 使用，需简洁明确。
- 不要把业务模型或 SQL 直接塞进工具里；工具只负责参数解析、调用领域能力、返回结果。

## 6. 当前已注册工具

| 工具名 | 模块 | 用途 |
|--------|------|------|
| retrieve_documents | retrieval | 内部知识库检索 |
| get_current_time | time_tools | 当前日期时间（可带时区） |
| echo | common | 原样返回文本（测试用） |
| list_db_tables | db | 列出数据库表名（只读） |
| query_examples | db | 查询 Example 列表（只读，调 domain 仓库） |

DB 相关工具在 `tools/db.py`，内部通过 `app.db.engine` 或 `app.domain.repositories` 访问数据，不直接放模型/仓库（详见 [04-数据库与存储](./04-数据库与存储.md)）。
