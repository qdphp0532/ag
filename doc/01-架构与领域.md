# 架构与领域

## 1. 总体分层

```
┌─────────────────────────────────────────────────────────────┐
│  HTTP 层（FastAPI）                                         │
│  main.py, api/routes, api/schemas, api/deps                 │
├─────────────────────────────────────────────────────────────┤
│  编排层（统一 Agent）                                        │
│  agents/agent.py（LangChain create_agent）                   │
├─────────────────────────────────────────────────────────────┤
│  能力层（工具与实现）                                        │
│  agents/tools/（按领域：retrieval, time_tools, common）     │
├─────────────────────────────────────────────────────────────┤
│  数据与存储                                                  │
│  domain/, db/                                                │
├─────────────────────────────────────────────────────────────┤
│  配置与基础设施                                              │
│  config.py, core/errors.py                                  │
└─────────────────────────────────────────────────────────────┘
```

- **HTTP 层**：路由、请求校验、依赖注入、异常处理；不包含业务与编排逻辑。
- **编排层**：统一使用 LangChain `create_agent`，对外只保留一个 `invoke` 入口。
- **能力层**：可被编排层调用的工具（`@tool`）及实现（如检索）；按领域放在 `agents/tools/`。
- **数据与存储**：`domain/` 放业务数据模型与仓库，`db/` 放连接与会话。
- **配置与基础设施**：环境配置、统一异常。

```
│   ├── db/                     # 数据库层：仅 base / engine / session（见 doc/04）
│   │   ├── __init__.py
│   │   ├── base.py
│   │   ├── engine.py
│   │   └── session.py

│   │   ├── models/
│   │   └── repositories/
│   ├── api/

│   │   ├── deps.py             # 配置、Agent 等依赖

│   │   └── schemas/

│   ├── conftest.py

├── requirements.txt

```

**领域划分**：

| 领域 | 位置 | 职责 |
|------|------|------|
| 应用入口 | `main.py` | 创建 FastAPI、生命周期、挂载 api_router |
| 配置 | `config.py` | 环境变量、LLM 参数、数据库 URL |
| 数据库 | `db/` | 仅引擎、会话、Base；详见 [04-数据库与存储](./04-数据库与存储.md) |
| 领域 | `domain/` | 模型（表）、仓库（数据访问）；依赖 db 的 Base 与 get_session |
| 基础设施 | `core/` | 异常类与全局处理 |
| HTTP API | `api/` | 路由、请求/响应模型、依赖注入 |
| 编排 | `agents/agent.py` | 统一 Agent 入口 |
| 工具 | `agents/tools/` | 按领域分文件的 @tool 及底层实现 |

## 3. 请求路径

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| POST | `/api/v1/invoke` | 调用统一的 LangChain Agent |

请求体为 `{"query": "..."}`，响应为 `{"answer", "route", "docs"}`。

## 4. 配置项

见 [00-快速开始](./00-快速开始.md) 中的环境变量表。配置由 `config.py` 的 `Settings` 统一加载，供 API 与 agents 共用。

## 5. 扩展点

- **新工具**：在 `agents/tools/` 增加模块或工具，并在 `tools/__init__.py` 注册到 `AGENT_TOOLS`（详见 [02-工具开发](./02-工具开发.md)）。
- **数据库与领域**：新表在 `domain/models/`，新仓库在 `domain/repositories/`；DB 工具在 `tools/db.py` 内调领域或 engine（详见 [04-数据库与存储](./04-数据库与存储.md)）。
- **复杂流程**：只有出现明确多步骤分支或循环后，再单独引入 LangGraph。
- **鉴权/限流**：在 `api/deps.py` 增加依赖，或在 `main.py` 挂中间件。
