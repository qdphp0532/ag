# 大模型多厂商配置

Agent 与工作流使用的大模型支持多厂商独立配置（OpenAI、通义千问、DeepSeek、豆包等），通过环境变量指定**默认厂商**及各厂商的 API Key、Base URL、模型名。

## 1. 设计思路

- **统一入口**：代码侧通过 `get_settings().get_llm_config(provider)` 或 `llm_kwargs(provider)` 获取某厂商的配置，再交给 `ChatOpenAI`（各厂商多为 OpenAI 兼容协议）。
- **默认厂商**：`DEFAULT_LLM_PROVIDER` 决定当前进程使用的厂商；不设则用 `openai`，与历史仅配 `OPENAI_*` 的行为一致。
- **按厂商隔离**：每个厂商一套独立 env（如 `QWEN_API_KEY`、`QWEN_BASE_URL`、`QWEN_MODEL`），互不覆盖，便于同时配置多个接入商、按需切换。

## 2. 支持的厂商与环境变量

| 厂商 | `DEFAULT_LLM_PROVIDER` 取值 | 环境变量 |
|------|-----------------------------|----------|
| OpenAI | `openai` | `OPENAI_API_KEY`, `OPENAI_BASE_URL`, `LLM_MODEL` |
| 通义千问 Qwen | `qwen` | `QWEN_API_KEY`, `QWEN_BASE_URL`, `QWEN_MODEL` |
| DeepSeek | `deepseek` | `DEEPSEEK_API_KEY`, `DEEPSEEK_BASE_URL`, `DEEPSEEK_MODEL` |
| 豆包 Doubao | `doubao` | `DOUBAO_API_KEY`, `DOUBAO_BASE_URL`, `DOUBAO_MODEL` |

- `*_BASE_URL`、`*_MODEL` 均为可选；不填时使用代码中的默认模型名（如 Qwen 默认 `qwen-plus`）。
- 若将 `DEFAULT_LLM_PROVIDER` 设为 `qwen`，则 Agent 与工作流都会使用 `get_llm_config(None)` → Qwen 的配置。

## 3. 配置示例

### 仅用 OpenAI（与之前一致）

```env
OPENAI_API_KEY=sk-xxx
# OPENAI_BASE_URL=...  可选
# LLM_MODEL=gpt-4.1-mini  可选，默认即此
```

### 默认用通义千问

```env
DEFAULT_LLM_PROVIDER=qwen
QWEN_API_KEY=sk-xxx
# QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1  部分环境需显式写
# QWEN_MODEL=qwen-plus  可选
```

### 默认用 DeepSeek

```env
DEFAULT_LLM_PROVIDER=deepseek
DEEPSEEK_API_KEY=sk-xxx
# DEEPSEEK_BASE_URL=  可选，官方默认即可
# DEEPSEEK_MODEL=deepseek-chat  可选
```

### 默认用豆包

```env
DEFAULT_LLM_PROVIDER=doubao
DOUBAO_API_KEY=xxx
# DOUBAO_BASE_URL=  按豆包文档填写
# DOUBAO_MODEL=doubao-pro-32k  可选
```

### 多厂商同时配置、切换默认

可在一份 `.env` 里同时配置多个厂商，只改 `DEFAULT_LLM_PROVIDER` 即可切换当前使用的厂商，无需改代码。

## 4. 代码使用方式

- **Agent / 工作流**：内部使用 `get_settings().llm_kwargs()`（不传参），即使用默认厂商；无需改现有调用即可支持多厂商。
- **按厂商取配置**：
  - `get_settings().get_llm_config("qwen")` → `LLMProviderConfig(api_key=..., base_url=..., model=...)`
  - `get_settings().llm_kwargs("deepseek")` → 传给 `ChatOpenAI(**kwargs)` 的字典。
- **扩展新厂商**：在 `app/config.py` 的 `Settings` 中增加该厂商的 `xxx_api_key`、`xxx_base_url`、`xxx_model`，并在 `get_llm_config()` 中增加对应分支；同时在 `LLM_PROVIDER_NAMES` 中加上新厂商名，便于文档与校验。

## 5. 注意事项

- 当前 **未** 支持「单次请求指定厂商」（如请求体里带 `provider: qwen`）；若需要，可在路由层读取请求参数，调用 `get_llm_config(provider)` 构造 LLM 再调用 Agent（需避免 get_agent 单例内写死厂商，可改为按 provider 缓存或每次从 config 取）。
- 各厂商 Base URL、模型名以官方文档为准；`BASE_URL` 为空时多数 SDK 使用官方默认地址。
