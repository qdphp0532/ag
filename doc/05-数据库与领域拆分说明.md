# 数据库与领域拆分说明

这份说明保留成一页摘要，只讲为什么这样拆；具体用法以 [04-数据库与存储](./04-数据库与存储.md) 为准。

## 1. 结论

当前项目把数据相关代码分成三层：

- `app/db/`：连接、引擎、Session。
- `app/domain/`：业务模型和仓库。
- `app/agents/tools/db.py`：给 Agent 暴露只读数据能力。

## 2. 这样拆的原因

- **减少耦合**：数据库连接和业务实体不是一回事，分开后依赖方向更清楚。
- **方便扩展**：以后加表、加仓库、加 Agent 数据工具时，改动范围更可控。
- **更容易读**：新人看到目录就能知道哪里放连接，哪里放业务，哪里放 Agent 工具。
- **更容易测**：仓库、工具、数据库初始化可以分别测，不会全绑在一起。

## 3. 额外约定

- 仓库方法统一由调用方传入 `Session`，不在仓库内部偷偷开连接。
- Agent 的 DB 工具只做只读查询，不直接写 SQL，不做写操作。
- 应用启动时先导入 `domain.models`，再执行 `init_db()`。

## 4. 什么时候看这份文档

- 只想快速理解为什么目录这样拆，可以先看这里。
- 要写代码、加模型、加仓库、加 DB 工具，直接看 [04-数据库与存储](./04-数据库与存储.md)。
