# 架构与领域

## 1. 总体分层

```
┌─────────────────────────────────────────────────────────────┐
│  HTTP 层（FastAPI）                                         │
│  main.py, api/routes, api/schemas, api/deps                 │
├─────────────────────────────────────────────────────────────┤
│  编排层（Agent / 工作流）                                    │
│  agents/agent.py（LangChain 工具循环）, agents/workflow_graph.py（LangGraph 图） │
├─────────────────────────────────────────────────────────────┤
│  能力层（工具与实现）                                        │
│  agents/tools/（按领域：retrieval, time_tools, common）     │
├─────────────────────────────────────────────────────────────┤
│  配置与基础设施                                              │
│  config.py, core/errors.py                                  │
└─────────────────────────────────────────────────────────────┘
```

- **HTTP 层**：路由、请求校验、依赖注入、异常处理；不包含业务与编排逻辑。
- **编排层**：LangChain Agent（工具循环）与 LangGraph 工作流（多节点图）；对外提供 `invoke`。
- **能力层**：可被编排层调用的工具（`@tool`）及实现（如检索）；按领域放在 `agents/tools/`。
- **配置与基础设施**：环境配置、统一异常。

## 2. 目录与领域

```
ag/
├── app/
│   ├── main.py                 # 应用入口、生命周期、挂载路由
│   ├── config.py               # Settings（pydantic-settings）
│   ├── core/
│   │   ├── __init__.py
│   │   └── errors.py           # AppException、全局异常处理
│   ├── api/
│   │   ├── __init__.py
│   │   ├── deps.py             # 配置、Agent、工作流图 等依赖
│   │   ├── routes/
│   │   │   ├── __init__.py     # 聚合路由
│   │   │   ├── health.py       # GET /health
│   │   │   └── agents.py       # POST /api/v1/invoke, POST /api/v1/workflow
│   │   └── schemas/
│   │       ├── __init__.py
│   │       └── agents.py       # InvokeRequest, InvokeResponse
│   └── agents/
│       ├── __init__.py
│       ├── agent.py            # LangChain Agent（bind_tools + 工具循环）
│       ├── workflow_graph.py   # LangGraph 工作流（检索 → 回答）
│       └── tools/              # 工具包（按领域）
│           ├── __init__.py     # AGENT_TOOLS、get_agent_tools、get_tools_by_name
│           ├── retrieval.py   # retriever() + retrieve_documents
│           ├── time_tools.py  # get_current_time
│           └── common.py      # echo
├── doc/                        # 研发文档（本目录）
├── tests/
│   ├── conftest.py
│   └── api/
│       └── test_health.py
├── requirements.txt
└── .env
```

**领域划分**：

| 领域 | 位置 | 职责 |
|------|------|------|
| 应用入口 | `main.py` | 创建 FastAPI、生命周期、挂载 api_router |
| 配置 | `config.py` | 环境变量、LLM 参数 |
| 基础设施 | `core/` | 异常类与全局处理 |
| HTTP API | `api/` | 路由、请求/响应模型、依赖注入 |
| 编排 | `agents/agent.py`, `agents/workflow_graph.py` | Agent 与工作流图 |
| 工具 | `agents/tools/` | 按领域分文件的 @tool 及底层实现 |

## 3. 请求路径

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| POST | `/api/v1/invoke` | 调用 LangChain Agent（工具循环） |
| POST | `/api/v1/workflow` | 调用 LangGraph 工作流（检索 → 回答） |

`invoke` 与 `workflow` 请求体均为 `{"query": "..." , "thread_id": "可选"}`，响应均为 `{"answer", "route", "docs"}`。

## 4. 配置项

见 [00-快速开始](./00-快速开始.md) 中的环境变量表。配置由 `config.py` 的 `Settings` 统一加载，供 API 与 agents 共用。

## 5. 扩展点

- **新工具**：在 `agents/tools/` 增加模块或工具，并在 `tools/__init__.py` 注册到 `AGENT_TOOLS`（详见 [02-工具开发](./02-工具开发.md)）。
- **新节点/新图**：在 `workflow_graph.py` 增加节点与边，或新建图模块并在 `api/deps.py` 暴露。
- **鉴权/限流**：在 `api/deps.py` 增加依赖，或在 `main.py` 挂中间件。
