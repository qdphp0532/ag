# 选型与扩展

## 1. LangChain 与 LangGraph 选型

- **LangChain**：适合单一目的、固定工具的 Agent（一问一答 + 工具循环），用 `create_agent` 或当前手写工具循环即可。
- **LangGraph**：适合需要自定义控制流的场景（多步分支、循环、人审、状态持久化）。

二者可混用：简单对话走 LangChain Agent（`/api/v1/invoke`），复杂流程走 LangGraph 图（`/api/v1/workflow`），底层能力（模型、检索、工具）统一用 LangChain/配置。

## 2. 混用方式

- **Agent**：`agents/agent.py`，`get_agent().invoke(...)`，工具来自 `agents/tools/`。
- **工作流**：`agents/workflow_graph.py`，`get_workflow_graph().invoke(...)`，节点内可使用 `get_settings()`、`tools.retrieval.retriever`、`ChatOpenAI` 等。

共用：`config`、`agents/tools/`（含 retrieval）、同一套 API 请求/响应格式。

## 3. 扩展建议

- **流式**：在 `agent.py` 或 workflow 中实现 stream，在 `api/routes/agents.py` 用 SSE/WebSocket 转发。
- **会话记忆**：在 `invoke_agent` 或图配置中根据 `thread_id` 接入 checkpointer / 历史存储。
- **人审**：在 LangGraph 节点中 `interrupt()`，并用 `Command(resume=...)` 恢复（参见 LangGraph 人审文档）。
- **新图**：新建图模块，在 `api/deps.py` 中暴露并增加对应路由。
